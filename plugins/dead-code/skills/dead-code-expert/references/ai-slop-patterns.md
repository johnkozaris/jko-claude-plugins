# AI Code Slop Patterns

Dead code fingerprints generated by AI coding assistants. They look professional -- good variable names, clear type annotations -- but they're dead weight.

## Pattern 1: Copy-Paste Proliferation

Identical logic blocks copied across multiple files instead of extracted into a shared function.

**Telltale signs:**
- Retry-with-backoff logic duplicated across 3+ services
- Error handling blocks identical in multiple controllers
- Validation logic repeated at every API endpoint
- Date/time formatting done differently in each file

**Detection:**
```bash
npx jscpd --min-lines 5 --threshold 5 .
```

**Fix:** Extract shared logic into a utility. Apply the DRY escalation ladder from `duplicate-code.md`.

## Pattern 2: Wrapper Functions That Add Nothing

Functions whose body is a single call to another function with the same arguments.

**Telltale signs:**
```python
def get_user(user_id):
    return user_repository.get_user(user_id)

def save_order(order):
    return order_service.save(order)
```

**Detection:**
```bash
rg -U '(def|function|fn) \w+\(.*\).*\n\s*(return )?\w+\.\w+\(' . --glob '!*test*'
```

**Fix:** Inline the wrapper. Re-introduce the layer only when it genuinely adds value (error handling, logging, caching, auth).

## Pattern 3: Speculative Interfaces and Abstractions

Interfaces, abstract classes, factory/strategy patterns with exactly one implementation.

**Telltale signs:**
- `IUserService` with only `UserService` implementing it (and no test mock)
- `AbstractPaymentProcessor` with only `StripePaymentProcessor`
- `OrderFactory.create()` that always returns the same type

**Detection:**
```bash
rg 'interface (\w+)' --type ts -o -r '$1' --no-filename | sort -u | while read iface; do
  count=$(rg "implements $iface" --type ts -c 2>/dev/null | awk -F: '{s+=$2} END {print s+0}')
  [ "$count" -le 1 ] && echo "Single-impl: $iface ($count)"
done
```

**Fix:** Delete the interface, rename the implementation. Re-introduce only when the second implementation actually exists.

## Pattern 4: Commented-Out "Previous Attempts"

Blocks of commented-out code left from AI iteration cycles.

**Telltale signs:**
- `// Old implementation` / `// Previous approach` / `// TODO: remove`
- Multiple commented blocks in the same function, each a different approach
- Comments containing complete function bodies

**Detection:**
```bash
rg -n '(//|#)\s*(old|previous|original|v[0-9]|attempt|try|backup|before)' . --glob '!*test*' -i
```

**Fix:** Delete all of it. Git has the history.

## Pattern 5: Excessive Inline Comments

Comments on every line that restate what the code does rather than why.

**Telltale signs:**
```javascript
// Initialize the user array
const users = [];
// Loop through the results
for (const result of results) {
  // Push the user to the array
  users.push(result);
}
// Return the users
return users;
```

**Detection:**
```bash
total_lines=$(rg -c '.' . --glob '*.{ts,js,py,rs}' --glob '!node_modules*' | awk -F: '{s+=$2} END {print s}')
comment_lines=$(rg -c '^\s*(//|#)' . --glob '*.{ts,js,py,rs}' --glob '!node_modules*' | awk -F: '{s+=$2} END {print s}')
echo "Comment ratio: $comment_lines / $total_lines"
```

**Fix:** Delete every comment that restates the code. Keep only comments that explain WHY.

## Pattern 6: Phantom Edge Case Handling

Defensive code for scenarios that cannot occur given the type system or calling context.

**Telltale signs:**
```typescript
// x is already non-nullable by type
if (x === null || x === undefined) { throw new Error("x is null"); }

// enum is exhaustive, this arm can't be reached
match color {
    Color::Red => ...,
    Color::Green => ...,
    Color::Blue => ...,
    _ => unreachable!(),
}
```

**Fix:** Remove checks the type system guarantees. Trust the compiler.

## Pattern 7: Reimplementing Standard Library

Hand-rolled logic for operations the standard library or installed packages already provide.

**Telltale signs:**
- Custom `debounce()` when lodash is installed
- Hand-rolled `groupBy()` instead of `Object.groupBy()`
- Custom date formatting instead of `Intl.DateTimeFormat`
- Custom retry logic instead of using a retry library

**Detection:**
```bash
rg 'function (debounce|throttle|deepClone|groupBy|flatten|uniq|chunk)\b' . --glob '!node_modules*'
```

**Fix:** Replace with the standard library or installed package equivalent. Delete the hand-rolled version.

## Pattern 8: Orphaned AI-Generated Files

Complete files generated by AI that were never wired into the application.

**Telltale signs:**
- Files with zero imports from anywhere in the project
- Utility files with functions that are never called
- Component files not referenced in any route or layout

**Detection:**
```bash
npx knip --include files
```

**Fix:** Delete files with zero references. Git has the history.

## Pattern 9: Refactoring Avoidance

New code added alongside old code without cleaning up the old. Two implementations coexist.

**Telltale signs:**
- `utils.js` and `helpers.js` with overlapping functions
- `UserService` and `UserServiceV2` both in use
- Functions with `_old`, `_new`, `_v2` suffixes

**Detection:**
```bash
rg '(v2|V2|_new|_old|_legacy|_deprecated|Old|New|Legacy)' . --glob '*.{ts,js,py,rs,go,swift,cs,java}' --glob '!node_modules*' --glob '!*test*' -l
```

**Fix:** Choose one implementation. Migrate all callers. Delete the other.

## Pattern 10: Security Theater in Tests

Tests that achieve high coverage but test nothing meaningful. Assertions that can never fail.

**Telltale signs:**
```javascript
test("should work", () => {
  const result = processOrder(mockOrder);
  expect(result).toBeDefined();  // always passes
  expect(result).not.toBeNull(); // always passes
});
```

**Detection:**
```bash
rg '(toBeDefined|not\.toBeNull|toBeTruthy|assert True|assertTrue\(true)' . --glob '*test*' --glob '*spec*'
```

**Fix:** Replace weak assertions with specific value checks. Delete tests that assert nothing.

## Pattern 11: Unnecessary Backward Compatibility

Re-exports, aliases, shims, adapter layers, and deprecated wrappers kept after a migration is complete. If the old path has zero consumers, the compat code is dead.

**Telltale signs:**
- Re-export files that exist only to redirect old import paths to new ones:
  ```python
  # old_module.py -- exists only for backward compat
  from new_module import *  # or specific re-exports
  ```
  ```typescript
  // old-path/index.ts
  export { UserService } from '../new-path/user-service';
  ```
- `@deprecated` / `#[deprecated]` annotations with no removal date
- Adapter/shim classes that translate old API to new with no callers using the old API
- `__all__` lists re-exporting moved symbols
- Compatibility imports in `__init__.py` for relocated modules
- `// backward compat` / `// legacy support` / `// migration shim` comments
- Package re-exports in barrel files after a refactoring

**Detection:**
```bash
# Find re-export/alias patterns
rg '(backward compat|legacy support|migration shim|compat|deprecated.*will be removed|re-export)' . -i --glob '!node_modules*' --glob '!target*'

# Find files that only re-export (Python)
rg -l '^from .+ import' . -t py | while read f; do
  lines=$(rg -v '(^#|^$|^from .+ import)' "$f" | wc -l)
  [ "$lines" -eq 0 ] && echo "Pure re-export file: $f"
done

# Find deprecated annotations without consumers
rg '(@deprecated|#\[deprecated|@Deprecated|Obsolete)' . --glob '!node_modules*' --glob '!target*' -l

# Find TypeScript barrel re-exports after migration
rg "export \{.*\} from '\.\." . --type ts -l
```

**Fix:** Check if anything still imports from the old path. If not, delete the shim. If the migration is complete and all consumers use the new path, every re-export and adapter is dead code. For libraries with external consumers, follow the deprecation cycle in `safe-removal.md`.

## The AI Slop Scorecard

| Check | Command | AI Slop Signal |
|---|---|---|
| Duplicate blocks | `npx jscpd --min-lines 5 .` | >5% duplication |
| Comment density | Count `//`/`#` lines vs total | >25% comments |
| Single-impl interfaces | Grep interfaces, count impls | >3 single-impl |
| Orphaned files | `npx knip --include files` | Any found |
| v2/old/legacy names | Grep for naming patterns | Any found |
| Weak test assertions | Grep for `toBeDefined` etc. | >10% of tests |
| Wrapper functions | Find single-line bodies | >5 found |
| Reimplemented stdlib | Grep for common function names | Any found |
| Security theater tests | Grep for `toBeDefined` etc. | >10% of tests |
| Compat shims/re-exports | Grep for "backward compat", pure re-export files | Any found |

**Score 0-2 signals: Clean.** Minimal AI debt.
**Score 3-5 signals: Moderate.** Focused cleanup needed.
**Score 6+: Heavy AI slop.** Run `/dead-code-clean aggressive`.
